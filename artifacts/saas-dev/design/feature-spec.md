# Feature Spec

- Gate-ready: yes
- Keyword: Image Size Checker
- Recommended MVP: requirement-aware Image Size Checker homepage plus a dedicated Image Resizer page. Both are client-side, private, batch-capable, and preset-driven.
- User-approved decisions: remove Requirement selector from checker; hide upload after result; show all Web/SEO, Social, and Print preset dimensions at the bottom of the result; add Image Resizer page; support batch upload and batch summary; first presets are Web/SEO, Social, Print; URL import later.

| Feature                        | MVP/later | Pain/gap/user request                                                     | Competitor comparison                                                 | Why it better satisfies need                                            | Acceptance criteria                                                                                |
| ------------------------------ | --------- | ------------------------------------------------------------------------- | --------------------------------------------------------------------- | ----------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| Upload-first checker           | MVP       | Requirement selector confused users and caused misleading Fail states     | Competitors start with upload, but do not show all-scenario diagnosis | Keeps the first action obvious and removes arbitrary requirement choice | Initial page top shows upload only; no Requirement selector before upload                          |
| Result-only state after upload | MVP       | User requested no upload component after results                          | Competitors often keep page static                                    | Gives attention to preview/info/results                                 | After at least one image uploads, upload dropzone is hidden and a New check action is available    |
| Image information panel        | MVP       | Users need file name, dimensions, file size, format, aspect ratio, pixels | Standard checker baseline                                             | Provides expected metadata in a polished layout                         | Result shows preview and image info for selected image                                             |
| All-preset fit report          | MVP       | Users need to know if the image works for Web/SEO/Social/Print            | Most competitors show only raw dimensions                             | Converts numbers into decisions and next actions                        | Web/SEO, Social, Print sections list preset target, actual delta, aspect guidance, action label    |
| Batch upload and batch summary | MVP       | Teams process multiple assets                                             | Resizer tools support batch; checker tools rarely do                  | Makes checker useful for asset folders                                  | Multiple files upload; summary shows count, dimensions, readiness counts, selected image switching |
| Print size calculations        | MVP       | Print dimensions require PPI math                                         | PosterBurner sends users to a guide                                   | Inline table prevents context switching                                 | Result shows 300/150/100/72 PPI rows with inches and cm                                            |
| Dedicated Image Resizer page   | MVP       | Users naturally need to fix dimensions after checking                     | Resizer competitors are separate from checker                         | One-stop workflow                                                       | `/resizer` page accepts images, presets, dimensions, format, quality, downloads                    |
| Preset-driven resizing         | MVP       | Users do not always know target sizes                                     | Bulk tools require target knowledge                                   | Preset categories reduce cognitive load                                 | Web/SEO/Social/Print presets populate width/height                                                 |
| Manual crop controls           | MVP       | Aspect-ratio changes need framing                                         | Many tools split crop into separate page                              | Keeps crop in resize workflow                                           | User can set crop area numerically/visually enough to crop selected image before output            |
| Download all ZIP               | MVP       | Batch outputs need one download                                           | Bulk resizers often make bulk export expected                         | Saves repeated downloads                                                | Generated resized files can be downloaded individually and as a ZIP                                |
| URL import/check               | Later     | User explicitly deferred URL                                              | Some tools support URL/cloud imports                                  | Avoids scope creep                                                      | No URL field in MVP, noted as future                                                               |
| SEO landing content            | MVP       | Skill requires SEO-friendly landing                                       | Competitors use long SEO content                                      | Captures search intent without hiding tool                              | Homepage visible text >=1000 words; density 3%-5%; metadata/canonical/H1 correct                   |
