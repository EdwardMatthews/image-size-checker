## Observed UX Lessons

- Replace hard "Fail" states with educational fit states. The user should understand the delta and the suggested action.
- Keep the initial screen focused on upload. The prior requirement selector created confusion because a correct image could appear to fail an arbitrary selected requirement.
- After upload, hide the upload component and treat the result as the primary state.
- Use batch summary like resizer tools, but add per-image diagnostic context that ordinary resizers lack.
- Include format validation and local-processing privacy reassurance near upload.
- Provide a direct bridge from checker result rows to the Image Resizer page.
