## Competitive Patterns

- Common checker pattern: one upload zone, then image metadata and SEO FAQ content.
- Common resizer pattern: upload first, then choose resize mode and output format/quality.
- Common trust promise: privacy/local processing is a repeated claim.
- Common missing layer: competitors rarely translate dimensions into "what this is good for" across Web/SEO, Social, and Print at once.
- Common batch gap: checker pages are usually single-image; resizer pages support batch but do not provide pre-resize analysis.

## Opportunity Summary

The better product is a requirement-aware checker: upload first, then show preview, metadata, print-size calculations, and all preset comparisons without forcing a requirement selector. The resizer should inherit the same preset model and let users fix the gaps discovered by the checker.
